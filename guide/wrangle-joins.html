Syntax

Like other siuba verbs, joins can be used in two ways: directly passing both data as arguments, or by piping.

# directly passing data
inner_join(lhs, rhs, on="id")

# piping
lhs >> inner_join(_, rhs, on="id")
id val_x val_y
0 1 lhs.1 rhs.1
1 2 lhs.2 rhs.2

Note that when used in a pipe, the first argument to the join must be _, to represent the data.

Match on multiple columns

lhs_multi = pd.DataFrame({
    "source": ["a", "a", "b"],
    "id": [1, 2, 1],
    "val": ["lhs.1", "lhs.2", "lhs.3"]
})

rhs_multi = pd.DataFrame({
    "source": ["a", "b", "c"],
    "id": [1, 1, 1],
    "val": ["lhs.1", "lhs.2", "lhs.3"]
})

inner_join(lhs_multi, rhs_multi, on=["source", "id"])
source id val_x val_y
0 a 1 lhs.1 lhs.1
1 b 1 lhs.3 lhs.2 
inner_join(lhs_multi, rhs_multi, on={"source": "source", "id": "id"})
source id val_x val_y
0 a 1 lhs.1 lhs.1
1 b 1 lhs.3 lhs.2

Match on expressions

SQL backends can join by expressions.

from sqlalchemy import create_engine
from siuba.sql import LazyTbl

engine = create_engine("sqlite:///:memory:")

lhs.to_sql("lhs", engine, index=False)
rhs.to_sql("rhs", engine, index=False)

tbl_sql_lhs = LazyTbl(engine, "lhs")
tbl_sql_rhs = LazyTbl(engine, "rhs")

inner_join(
    tbl_sql_lhs,
    tbl_sql_rhs,
    sql_on = lambda lhs, rhs: lhs.val <= rhs.val
)
# Source: lazy query
# DB Conn: Engine(sqlite:///:memory:)
# Preview:
id_x val_x val_y id_y
0 1 lhs.1 rhs.1 1
1 1 lhs.1 rhs.2 2
2 1 lhs.1 rhs.3 4
3 2 lhs.2 rhs.1 1
4 2 lhs.2 rhs.2 2

# .. may have more rows
